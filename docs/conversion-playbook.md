# Conversion Playbook

Conversion improvements should clarify the user's next step without creating pressure or confusion.

## Primary CTA Rules

- One primary CTA per page section.
- Use action language tied to the user goal.
- Keep CTA labels specific, such as `Book a Launch Audit` or `Request a Performance Review`.
- Repeat the primary CTA after major proof or value sections.
- Make contact and trust information visible before asking for commitment.

## Trust Signals

Use real evidence:

- Case studies.
- Metrics with context.
- Testimonials with names and roles when permitted.
- Security, performance, or SEO audit examples.
- Clear process and deliverables.
- Public GitHub activity where it supports credibility.

## Friction Checks

- Forms ask only for necessary information.
- Error messages explain how to recover.
- Mobile forms are easy to complete.
- CTA destinations match user expectations.
- Pricing, scope, or next steps are not hidden when they matter to the decision.

## Measurement

Track:

- CTA clicks.
- Form starts.
- Form completions.
- Booking completions.
- Scroll depth on long pages.
- Exit points in critical flows.

Do not collect unnecessary personal data.
