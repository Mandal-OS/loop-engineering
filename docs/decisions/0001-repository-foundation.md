# 0001 Repository Foundation

Date: 2026-06-27

## Status

Accepted

## Context

The repository started empty except for Git metadata. The first goal is to create a professional GitHub-ready foundation for Loop Engineering before application code is added.

## Decision

Use a docs-first repository foundation with:

- Production standards in `docs/`.
- Agent instructions in `AGENTS.md`.
- GitHub collaboration templates.
- Local and CI repository checks.
- Placeholder directories for future apps, packages, and audits.

## Consequences

- The repo can be published immediately without pretending unfinished application code exists.
- Future code has clear homes.
- Launch, security, performance, SEO, and conversion standards are documented from day one.
